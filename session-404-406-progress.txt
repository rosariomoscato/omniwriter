### Feature #404: Modello dati per quota spazio utente (100 MB)

**Objective:** Aggiungere al database il tracciamento dello spazio utilizzato per ogni utente. Ogni utente ha un limite di 100 MB.

**Implementation:**

#### Database Schema Changes

Added two new columns to the users table:
- storage_used_bytes (INTEGER, default 0) - Tracks total storage used
- storage_limit_bytes (INTEGER, default 104857600) - 100MB limit per user

Migration applied via ALTER TABLE statements and backfilled from existing sources table.

#### Helper Functions Created (server/src/utils/storage.ts)

1. increaseUserStorage(userId, bytes) - Increases storage_used_bytes after file upload
2. decreaseUserStorage(userId, bytes) - Decreases storage_used_bytes after file deletion (never below 0)
3. getUserStorageInfo(userId) - Returns { used, limit } in bytes
4. recalculateUserStorage(userId) - Recalculates from actual sources table
5. hasStorageQuota(userId, newFileBytes) - Checks if user has space for new file

#### Integration with File Operations

Modified server/src/routes/sources.ts:
- Import storage utility functions (Feature #404)
- Call increaseUserStorage after file upload (4 locations)
- Call decreaseUserStorage after file deletion (2 locations)

**What Counts Toward Storage:**
- Uploaded source files (PDF, DOCX, TXT, RTF) - stored in sources.file_size
- EPUB cover images
- Saved export files on server

**What Does NOT Count:**
- Generated text content (novels, essays, articles)

**Test Results:**
- Database columns created correctly
- Default limit is 100MB (104857600 bytes)
- Helper functions work correctly
- Integration with upload/delete verified
- Backfill from existing sources works
- Never goes below 0 on deletion
- Quota checking works

### Feature #406: API endpoint stato spazio utente

**Objective:** Creare un endpoint API che restituisca lo stato dello spazio dell'utente: spazio usato, spazio totale, spazio disponibile, percentuale di utilizzo.

**Implementation:**

#### API Endpoint Created

**Route:** GET /api/users/storage

**Authentication:** Required (uses authenticateToken middleware)

**Response Format:**
{
  "storage": {
    "used_bytes": 0,
    "limit_bytes": 104857600,
    "used_mb": 0.0,
    "limit_mb": 100.0,
    "percent_used": 0.0,
    "available_bytes": 104857600
  }
}

#### Implementation Details (server/src/routes/users.ts, lines 351-382)

Uses getUserStorageInfo helper to get data, then calculates:
- used_mb and limit_mb (rounded to 2 decimals)
- percent_used (rounded to 2 decimals)
- available_bytes (limit - used, never negative)
