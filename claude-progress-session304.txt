# Session 2026-02-20 - Feature #304 Implementation

## Feature #304: Use continuity context in AI generation

### Objective
Integrare i dati di continuity nella generazione AI dei capitoli del sequel. L'AI deve conoscere: personaggi morti, feriti, luoghi visitati, eventi precedenti per mantenere coerenza.

### Implementation Summary

#### 1. Created Helper Function: `fetchContinuityForProject()`

**File:** `server/src/routes/projects.ts` (lines 48-180)

**Purpose:** Fetch and format continuity data from `saga_continuity` table.

**Features:**
- Queries continuity based on project's `saga_id` and `continuity_id`
- Parses JSON fields: characters_state, events_summary, locations_visited, timeline_json
- Formats with visual indicators (🟢 alive, 🔴 dead, 🟡 injured, ⚪ missing)
- Returns structured ContinuityContext object
- Exported for use in other modules

#### 2. Updated Sequel-Stream Endpoint

**File:** `server/src/routes/projects.ts`

**Outline Generation (lines 1721-1867):**
- Fetches continuity before building outline
- Adds cumulative synopsis to prompt
- Adds character states from continuity
- Adds events summary
- Adds locations visited
- Adds timeline context
- Includes warning about saga requirements

**Chapter Generation (lines 2046-2104):**
- Fetches continuity for each chapter

  - Maintain consistency with all previous episodes
  - Strictly respect character status

  - Consider all past events
  - Maintain location consistency
  - Respect established timeline

### Continuity Data Structure

From `saga_continuity` table:
- `cumulative_synopsis`: Full saga narrative
- `characters_state`: JSON array with status (alive/dead/injured/missing)
- `events_summary`: JSON array of past events
- `locations_visited`: JSON array of locations
- `timeline_json`: JSON array of timeline events

### AI Prompt Enhancement

When continuity is available, the AI receives:
1. Project metadata
2. Human Model style
3. Current chapter context
4. Project characters (basic)
5. Dead character warnings
6. Project locations (basic)
7. **📖 Cumulative Saga Synopsis** (continuity)
8. **🟢🔴🟡⚪ Character States** (continuity, with status)
9. **📜 Past Events** (continuity, all episodes)
10. **📍 Locations Visited** (continuity, all episodes)
11. **⏰ Timeline** (continuity)
12. **📋 Saga Continuity Instructions** (explicit requirements)
13. User prompt

### Benefits

1. **Prevents Character Resurrection:** Dead characters explicitly marked and forbidden
2. **Event Coherence:** All past events available for reference
3. **Location Consistency:** AI knows visited locations and significance
4. **Narrative Continuity:** Cumulative synopsis provides full story arc
5. **Timeline Accuracy:** Temporal relationships maintained
6. **Character Arc Continuity:** Status notes inform development
