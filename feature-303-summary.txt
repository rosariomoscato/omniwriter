### Feature #303: Create Sequel with Continuity button

**Objective:** Nella pagina Saga, aggiungere opzione 'Crea Seguito' che usa la continuity per pre-popolare il nuovo progetto con personaggi e luoghi degli episodi precedenti.

**Implementation:**

#### 1. Created CreateSagaSequelModal Component
**File:** client/src/components/CreateSagaSequelModal.tsx

- Modal dialog for creating sequels at the saga level
- Form fields: title, description (optional), source project selection
- Shows preview of what will be imported (characters, locations)
- Auto-selects most recent project as source
- Displays warning if no projects exist in saga
- Calls `apiService.createSagaSequel()` endpoint
- Shows success message with counts of imported data
- Navigates to new project after creation
- Triggers reload of saga projects via onSuccess callback

#### 2. Updated SagasPage
**File:** client/src/pages/SagasPage.tsx

- Added import: `CreateSagaSequelModal` and `Sparkles` icon
- Added state: `showCreateSequelModal`
- Added "Create Sequel" button next to "Add Project" button
- Button only shows when `sagaProjects.length > 0`
- Uses purple color to differentiate from blue "Add Project" button
- Has Sparkles icon to indicate AI/continuity feature
- Added modal rendering at end of component

#### 5. Backend Endpoint (Already Existed)
**File:** server/src/routes/sagas.ts (lines 341-572)

The endpoint `POST /api/sagas/:id/create-sequel` was already implemented in Feature #300:
- Creates new project in saga
- Uses continuity data if available
- Copies characters (excluding dead ones)
- Copies locations
- Falls back to source project data if no continuity
- Returns sequel info with counts
