### Feature #401: Rimozione ruoli free_user, premium_user e lifetime_user

**Objective:** Remove completely the free_user, premium_user, and lifetime_user roles from the system. Replace them with only two roles: 'user' (normal user) and 'admin' (administrator). Update the role enum/constants, the User model in the database, and all references in the code.

**Implementation:**

#### 1. Database Migration
- Created migration script to convert existing roles:
  - All 'free', 'premium', and 'lifetime' users → 'user'
  - 'admin' users remain 'admin'
- Updated users table schema:
  - Changed CHECK constraint from `CHECK(role IN ('free', 'premium', 'lifetime', 'admin'))` to `CHECK(role IN ('user', 'admin'))`
  - Changed default from 'free' to 'user'
- Migration tested successfully on 57 existing users

#### 2. Backend Code Changes
- **server/src/config/tier-permissions.ts**: Updated to only have 'user' and 'admin' roles
  - All limits set to unlimited (null or -1)
  - All features enabled (true)
  - Both user and admin have identical full access

- **server/src/middleware/roles.ts**: Updated requirePremium middleware
  - Now allows any authenticated user (user or admin) through
  - Kept for backward compatibility but deprecated

- **server/src/routes/ai.ts**: Updated AI model tier metadata
  - Changed all model tiers from 'premium'/'free' to 'user'

- **server/src/routes/export.ts**: Removed premium checks
  - Removed 3 instances of premium format checks
  - All export formats (EPUB, PDF, etc.) now available to all users

#### 3. Frontend Code Changes
- **client/src/config/tier-config.ts**: Updated to two-role system
  - TIER_LIMITS now only has 'user' and 'admin'
  - Both have identical FULL_ACCESS_LIMITS
  - PREMIUM_TIERS includes both 'user' and 'admin' (all users are "premium")

- **client/src/hooks/useTierPermissions.ts**: Maps legacy roles to 'user'
  - Automatically converts 'free', 'premium', 'lifetime' to 'user' for compatibility

- **client/src/pages/SettingsPage.tsx**: Simplified tier display
  - Removed complex tier comparison logic
  - Shows simple "User" or "Admin" badge
  - Shows "Full Access Active" status for all users

- Created stub components:
  - **client/src/components/PremiumBadge.tsx**: Returns null (deprecated)
  - **client/src/components/UpgradeModal.tsx**: Returns null (deprecated)
