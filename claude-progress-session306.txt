## Feature #306: Fix character role/status extraction in continuity

### Problem Statement:
The continuity system was not correctly extracting character roles and statuses. In Novel 2, Marco was the King of Valoria, but in the sequel (Novel 3), he was presented as "capo delle guardie" (guard captain) and the King of Valoria was said to be dead.

### Root Cause Analysis:
1. The AI prompt for finalize-episode was extracting generic roles (e.g., "protagonista") instead of specific titles (e.g., "Re di Valoria")
2. The notes field contained important role change info but wasn't being prominently displayed
3. Users had no way to review and correct extracted continuity data before creating a sequel

### Implementation:

#### 1. Enhanced AI Prompt (server/src/routes/projects.ts, lines 6996-7070)
- Added explicit instructions for extracting SPECIFIC roles/titles (e.g., "King of Valoria", "Captain of the Royal Guard")
- Added warning examples showing correct vs incorrect role extraction
- Emphasized that role should reflect the FINAL state at the end of the episode
- Both Italian and English prompts updated

#### 2. Improved Continuity Formatting (server/src/routes/projects.ts, lines 134-165)
- Modified fetchContinuityForProject to display role and notes more prominently
- Added formatCharacterWithRole helper that shows: Name [Role] - Notes
- Added support for characters with "unknown" status that have role/notes
- Both Italian and English formatting

#### 3. New PUT Endpoint (server/src/routes/projects.ts, lines 7314-7390)
- Added PUT /api/projects/:id/continuity to update continuity data
- Accepts synopsis, characters, events, locations
- Updates both saga_continuity table AND characters table
- Returns updated continuity record

#### 4. Frontend API Method (client/src/services/api.ts, lines 2000-2025)
- Added updateContinuity(projectId, data) method
