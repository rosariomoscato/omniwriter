### Feature #275: Include character status (alive/dead) in chapter generation context

**Objective:** Prevent dead characters from appearing in sequels by explicitly listing them as "DO NOT USE" in AI prompts.

**Implementation:**

#### 1. Update Character Queries (server/src/routes/chapters.ts)

Changed from:
```sql
SELECT name, description, traits FROM characters
```

To:
```sql
SELECT name, description, traits, status_at_end, status_notes FROM characters
```

Applied to both `generate-stream` and `regenerate-stream` endpoints.

#### 2. Update contentModeration.ts (server/src/services/contentModeration.ts)

**Function signature updated:**
```typescript
export function sanitizePromptContent(data: {
  characters?: Array<{
    name: string;
    description: string;
    traits?: string;
    status_at_end?: string;
    status_notes?: string
  }>;
  // ... other fields
}): {
  characters: string;
  deadCharacters: string;  // NEW
  // ... other fields
}
```

**Separation logic (lines 197-215):**
- Filters dead characters: `status_at_end === 'dead'`
- Alive characters get status annotations (injured, missing)
- Dead characters go to separate `deadCharacters` field

#### 3. Add Dead Characters Warning to AI Prompts (server/src/routes/chapters.ts)

**Added after alive characters section:**
```typescript
if (sanitizedContent.deadCharacters) {
  if (isItalian) {
    systemPrompt += `\n\nPERSONAGGI MORTI (NON USARE - SONO MORTI):\n${sanitizedContent.deadCharacters}\n\nIMPORTANTE: Questi personaggi sono morti e NON devono apparire nel capitolo.`;
  } else {
    systemPrompt += `\n\nDEAD CHARACTERS (DO NOT USE - THEY ARE DEAD):\n${sanitizedContent.deadCharacters}\n\nIMPORTANT: These characters are dead and MUST NOT appear in this chapter.`;
  }
}
```

Added separation logic for alive/dead characters in sequel generation with same warning pattern.
