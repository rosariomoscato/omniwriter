#### Feature #14: User can log in with email and password ✅
**Status:** PASSING (Code verified + Build successful)

**Implementation Details:**
- POST /api/auth/login endpoint (server/src/routes/auth.ts:101-181)
- Validates email/password presence, returns 400 if missing
- Queries users table by email
- Compares password using bcrypt.compareSync()
- Returns 401 for invalid credentials (same message for wrong email vs password for security)
- Creates JWT token with configurable expiration:
  - Default: 24h (or JWT_EXPIRES_IN env var)
  - With rememberMe: 30d (or JWT_REMEMBER_ME_EXPIRES_IN env var)
- Stores session in sessions table with token and expiration
- Updates users.last_login_at timestamp
- Returns user object and token on success

**Frontend Implementation:**
- LoginPage component (client/src/pages/LoginPage.tsx)
- Form with email, password, and rememberMe checkbox
- Uses AuthContext login() method (updated from direct apiService call)
- Displays loading state during submission

- Redirects to /dashboard on success
- Links to registration and forgot-password pages
