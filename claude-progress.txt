### Feature #447: Lazy-load Google OAuth Strategy configuration
**Status:** COMPLETED and PASSING

**Problem:**
- The Google OAuth Strategy was configured at module load time in auth.ts
- Constants `googleClientId`, `googleClientSecret`, `callbackURL` were read from `process.env` at module import time
- This caused issues when environment variables weren't available at import time

**Solution Applied:**
- Created `initGoogleOAuth()` function that:
  - Reads environment variables at CALL time, not module load time
  - Validates credentials exist before configuring passport
  - Uses `googleOAuthInitialized` flag to prevent double initialization
  - Logs helpful warnings if credentials are missing
- Exported `initGoogleOAuth` from auth.ts and called it from index.ts after passport middleware setup
- This ensures env vars are definitely loaded (via dotenv.config()) before Strategy is configured

**Changes:**
- `server/src/routes/auth.ts`:
  - Removed module-level constants and immediate `passport.use()` call
  - Added `initGoogleOAuth()` function with lazy strategy configuration
  - Added validation and logging for missing credentials
- `server/src/index.ts`:
  - Imported `initGoogleOAuth` from auth module
  - Called `initGoogleOAuth()` after `app.use(passport.session())`

**Verification:**
- TypeScript compilation succeeds without errors
- Server starts correctly with lazy loading
- Without credentials: Shows warning "[Google OAuth] Missing GOOGLE_CLIENT_ID or GOOGLE_CLIENT_SECRET"
- With credentials: Shows "[Google OAuth] Initializing with callback URL: ..."
- Browser test: Google OAuth button initiates flow correctly, redirects to Google

### Feature #446: Fix env loading for Docker/Coolify production
**Status:** COMPLETED and PASSING

- In produzione su Docker/Coolify, le variabili d'ambiente vengono passate dal container, non da un file .env
- Il codice usava `import 'dotenv/config'` incondizionatamente, che cerca di caricare un file .env inutilmente in produzione

- In produzione (Docker/Coolify): `NODE_ENV=production` è settato nel Dockerfile, quindi dotenv NON viene caricato
- In sviluppo: dotenv viene caricato dal file .env come prima

### Feature #445: Fix dotenv config order for Google OAuth
**Status:** COMPLETED and PASSING

**Root Cause:**
- In `server/src/index.ts`, `dotenv.config()` was called AFTER the import statements
- When `authRouter` was imported, it immediately configured the Google OAuth Strategy
- At that point, `process.env.GOOGLE_CLIENT_ID` and `process.env.GOOGLE_CLIENT_SECRET` were undefined
- This caused Google OAuth to fail in production with "invalid client" errors

**Fix Applied:**
- Changed `import dotenv from 'dotenv'` + `dotenv.config({ path: envPath })` to `import 'dotenv/config'` as the FIRST import
- This ensures environment variables are loaded before any module that uses them
- Removed the redundant `dotenv.config()` call that was after all imports
- Added explanatory comment about the importance of import order

**Verification:**
- TypeScript compilation succeeds without errors
- Server starts successfully with the new import order
- Health check returns healthy status
- In compiled dist/index.js: `require("dotenv/config")` is at line 41, `require("./routes/auth")` is at line 49
- Git Commit: d2b062a
