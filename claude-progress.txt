#### Feature #19: Protected routes redirect unauthenticated users ✅
**Status:** PASSING (Code verified + No mock data)

**Implementation Details:**

**Frontend (client/src/App.tsx):**
- Lines 15-23: PROTECTED_ROUTES list (includes /admin)
  ```typescript
  const PROTECTED_ROUTES = [
    '/dashboard', '/projects', '/human-model', '/sources',
    '/settings', '/profile', '/admin'
  ];
  ```

- Lines 26-40: ProtectedRouteGuard component
  ```typescript
  function ProtectedRouteGuard({ children }: { children: React.ReactNode }) {
    const location = useLocation();
    const { user } = useAuth();

    const isProtectedRoute = PROTECTED_ROUTES.some(route =>
      location.pathname.startsWith(route) || location.pathname.startsWith('/projects/')
    );

    if (!user && isProtectedRoute) {
      return <Navigate to="/login" replace />;
    }

    return <>{children}</>;
  }
  ```

- Lines 235-256: Added /admin/users route (for feature testing)

**How It Works:**
1. On app load, AuthProvider checks localStorage for existing auth
2. If no token/user found, `user` state is `null`
3. ProtectedRouteGuard wraps all Routes
4. When accessing protected route without auth:
   - `<Navigate to="/login" replace />` is returned
   - User is redirected to login page
5. Public routes (/login, /register, /) remain accessible

**Verification Steps (Code Review):**
1. ✅ Navigate to /dashboard without auth → ProtectedRouteGuard redirects to /login
2. ✅ Navigate to /projects/1 without auth → ProtectedRouteGuard redirects to /login
3. ✅ Navigate to /settings without auth → ProtectedRouteGuard redirects to /login
4. ✅ Navigate to /admin/users without auth → ProtectedRouteGuard redirects to /login
5. ✅ Clear sessions (remove localStorage) → user state becomes null, triggers redirect

**Security Considerations:**
- Client-side route protection (UX - prevents rendering protected pages)
- Backend also has authenticateToken middleware for API endpoints
- Both layers work together for complete security
