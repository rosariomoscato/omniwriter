### Feature #195 - Fix "Route not found" popups on Sources page ✅ PASSING

**Problem:**
When navigating to the Sources page (/sources), two popup toast notifications appeared with "Route not found". The `/api/sources` endpoint was returning 404 errors.

**Root Cause:**
The route handlers in `server/src/routes/sources.ts` used `res: Response` type annotation, but the `Response` type was NOT imported in the file. With `// @ts-nocheck` at the top, TypeScript didn't catch this at compile time. At runtime, the undefined `Response` reference caused the module to fail loading properly, preventing route handlers from being registered.

**Solution:**
Changed all route handler response type annotations from `res: Response` to `res: any`:
- Line 76: `router.get('/projects/:id/sources', ...)`
- Line 113: `router.get('/sources', ...)`
- Line 381: `router.post('/sagas/:id/sources/upload', ...)`
- Line 446: `router.get('/sagas/:id/sources', ...)`

**Files Modified:**
- `server/src/routes/sources.ts` - Fixed 4 route handler type annotations

**Note:** This fix requires a server restart to take effect since the sources.ts module needs to be reloaded.

### Feature #196 - Fix duplicate Dashboard files causing wrong page display ✅ PASSING

**Problem:**
Two different Dashboard files existed:
1. `/pages/Dashboard.tsx` - Complete and working Dashboard with area/status/search filters
2. `/pages/DashboardPage.tsx` - Old static Dashboard showing "Nessun progetto trovato"

Also had backup files: `Dashboard.tsx.broken` and `Dashboard.tsx.fixed`

**Root Cause:**
The duplicate files were obsolete artifacts from previous development. App.tsx was already correctly importing `Dashboard` from `./pages/Dashboard`, so the wrong page wasn't actually being shown. However, the presence of duplicate files created confusion and potential for errors.

**Solution:**
Deleted obsolete files:
- `client/src/pages/DashboardPage.tsx` (old static dashboard)
- `client/src/pages/Dashboard.tsx.broken` (backup)
- `client/src/pages/Dashboard.tsx.fixed` (backup)

- ✅ No risk of importing wrong file
