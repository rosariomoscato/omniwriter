**Feature #184 - Google Drive save and load:** ✅ IMPLEMENTED
- Updated `server/package.json`:
  - Added `googleapis`: "^144.0.0"
  - Note: google-auth-library is part of googleapis

- Updated `server/src/db/database.ts`:
  - Added `google_access_token TEXT` column to users table
  - Added `google_refresh_token TEXT` column to users table
  - Tokens stored for Drive API access

- Updated `server/src/routes/auth.ts`:
  - Modified Google OAuth strategy callback to store tokens:
    - `accessToken` and `refreshToken` passed from Google
    - Existing users: UPDATE tokens on login
    - Linking accounts: UPDATE with tokens
    - New users: INSERT with tokens
  - Tokens stored for later use with Drive API

- Updated `server/src/routes/export.ts`:
  - Added imports: `google` from 'googleapis', `OAuth2Client` from 'google-auth-library'
  - Added helper `getDriveClient(user)`: Creates OAuth2 client with token refresh
  - Added helper `exportProjectAsBuffer(projectId, format)`: Generates export as buffer
  - Added POST `/api/projects/:id/google-drive/save`:
    - Requires premium (via requirePremium middleware)
    - Checks for Google tokens, returns 400 if not connected
    - Exports project in specified format (docx/epub/txt)
    - Uploads to Google Drive using drive.files.create
    - Returns file metadata (id, name, viewUrl, downloadUrl)
  - Added POST `/api/projects/:id/google-drive/load`:
    - Requires premium
    - Accepts fileId from request body
    - Downloads file from Google Drive
    - Detects file type from metadata
    - Parses using mammoth (docx) or parseTxtContent (txt)
    - Deletes existing chapters from project
    - Creates new chapters from imported content
    - Updates project word count
  - Added GET `/api/google-drive/files`:
    - Lists user's Drive files
    - Returns file metadata for selection UI
    - Ordered by modifiedTime desc

**Authentication Flow:**
1. User logs in with Google OAuth
2. access_token and refresh_token stored in users table
3. Drive API client initialized with tokens
4. Token refresh handled automatically by OAuth2 client

**Requirements:**
- Premium subscription enforced (requirePremium middleware)
- Google account must be connected
- Environment variables needed:
  - GOOGLE_CLIENT_ID
  - GOOGLE_CLIENT_SECRET
  - GOOGLE_CALLBACK_URL

**Database Migration:**
```sql
ALTER TABLE users ADD COLUMN google_access_token TEXT;
ALTER TABLE users ADD COLUMN google_refresh_token TEXT;
```
