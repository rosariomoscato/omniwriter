### Feature #181 - Character relationship mapping ✅ PASSING

**Implementation Summary:**
Created a visual relationship map component for Romanziere projects displaying character connections and allowing editing relationships.

**Changes Made:**

1. **Created RelationshipMap Component** (client/src/components/RelationshipMap.tsx):
   - Visual node-based graph with characters as circular nodes
   - SVG-based edge layer showing relationships
   - Circular layout auto-arranges characters
   - Color-coded relationships (family=blue, friend=green, enemy=red, etc.)
   - Interactive clicking to add relationships
   - Edge clicking to view details
   - Add relationship dialog
   - Color-coded legend
   - Full dark mode support

2. **Updated ProjectDetail.tsx**:
   - Added `showRelationshipMap` state
   - Added "Relationship Map" button (only shows with 2+ characters)
   - Added `handleAddRelationship()` handler
   - Integrated RelationshipMap component
   - Added Network icon import

**Relationship Data Structure:**
```json
[
  {
    "characterId": "uuid-of-source",
    "relatedCharacterId": "uuid-of-target",
    "relationshipType": "family|friend|enemy|romantic|mentor|ally|other"
  }
]
```
Stored in character's `relationships_json` field.

**Relationship Types & Colors:**
- Family: Blue (bg-blue-500)
- Friend: Green (bg-green-500)
- Enemy: Red (bg-red-500)
- Romantic: Pink (bg-pink-500)
- Mentor: Purple (bg-purple-500)
- Ally: Cyan (bg-cyan-500)
- Other: Gray (bg-gray-500)

**User Flow:**
1. User creates 2+ characters in Romanziere project
2. "Relationship Map" button appears in Characters section
3. Click button → visual map opens
4. Characters displayed as nodes in circular layout
5. Existing relationships shown as colored lines
6. Click character node → add relationship dialog
7. Select "To Character" and "Relationship Type"
8. Save → new edge appears, relationships_json updated, toast shown
9. Click edge to view relationship details
10. Map is persistent (reopens with all relationships)
