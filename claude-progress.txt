### Feature #222: Fix upload fonti standalone - file non salvati nel database

**Problem:** When uploading files from the standalone Sources page (/sources), uploads would sometimes fail with "UNIQUE constraint failed: sources.id" error.

**Root Cause Analysis:**
- The source ID generation used `Buffer.from(\`${userId}-${Date.now()}\`).toString('base64').slice(0, 36)`
- This approach had two issues:
  1. `Date.now()` only has millisecond precision
  2. Truncating to 36 characters could cause collisions
- When multiple uploads happened rapidly (same millisecond), duplicate IDs were generated

**Fix:**
1. Added `import { v4 as uuidv4 } from 'uuid';` to sources.ts
2. Changed all 4 source ID generation points to use `uuidv4()` instead of base64-encoded timestamp

**Files Changed:**
- `server/src/routes/sources.ts`:

  - Line 189: Standalone source upload ID

| System Prompt | Extracted to `system` field | Kept in messages array |
| Response Format | content[].text | choices[].message.content |
| Usage Format | input_tokens/output_tokens | prompt_tokens/completion_tokens |
| Stop Param | stop_sequences | stop |

**Implementation:**

1. **RequestyProvider.ts - Complete Rewrite:**
   - Changed `getDefaultBaseUrl()` to return `'https://router.requesty.ai/v1'`
   - Changed `getHeaders()` to use `Authorization: Bearer` (removed x-api-key, anthropic-version)
   - Changed `buildRequestBody()` to OpenAI format (keep system in messages, use `stop` instead of `stop_sequences`)
   - Changed `parseResponse()` to OpenAI format (extract from choices[].message)
   - Changed streaming to handle `[DONE]` marker and OpenAI delta format
