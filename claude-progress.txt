#### Feature #14: User can log in with email and password ✅
**Status:** PASSING (Code verified + Build successful)

**Implementation Details:**
- POST /api/auth/login endpoint (server/src/routes/auth.ts:101-181)
- Validates email/password presence, returns 400 if missing
- Queries users table by email
- Compares password using bcrypt.compareSync()
- Returns 401 for invalid credentials (same message for wrong email vs password for security)
- Creates JWT token with configurable expiration:
  - Default: 24h (or JWT_EXPIRES_IN env var)
  - With rememberMe: 30d (or JWT_REMEMBER_ME_EXPIRES_IN env var)
- Stores session in sessions table with token and expiration
- Updates users.last_login_at timestamp
- Returns user object and token on success

**Frontend Implementation:**
- LoginPage component (client/src/pages/LoginPage.tsx)
- Form with email, password, and rememberMe checkbox
- Uses AuthContext login() method (updated from direct apiService call)
- Displays loading state during submission
- Error display for failed login
- Redirects to /dashboard on success
- Links to registration and forgot-password pages

**AuthContext Integration:**

- Properly sets user state and token in context
- Calls ApiService.setAuth() to persist to localStorage
- Consistent with registration flow

**API Service:**

#### Feature #74: Upload writings for style analysis ✅
**Status:** PASSING (Code verified + No mock data)

**Implementation Details:**

**Backend (server/src/routes/human-models.ts):**
- POST /api/human-models/:id/upload - Upload file for analysis
  - Validates model ownership (user_id check)
  - Accepts: file_name, file_type, content_text
  - Counts words in content_text (splits on whitespace)
  - Saves file to /server/data/human-model-sources/
  - Creates directory if not exists
  - Inserts into human_model_sources table
  - Updates model's total_word_count (incremental)
  - Returns created source + new total word count
- POST /api/human-models/:id/analyze - Start style analysis
  - Validates minimum word count:
    - romanziere_advanced: 50,000 words
    - saggista_basic/redattore_basic: 10,000 words
  - Returns 400 if insufficient words
  - Updates status to 'analyzing'
  - Simulates AI analysis (2s timeout then status='ready')
- GET /api/human-models/:id/analysis - Get analysis results
  - Returns training_status + analysis_result_json

**Frontend (client/src/pages/HumanModelPage.tsx):**
- Click model in left panel to view details
- "Upload Writings" button opens upload dialog
- Upload dialog:
  - File input accepts .txt files
  - FileReader reads file content as text
  - Preview textarea shows file content
  - Word count display updates as content changes
  - Upload button disabled until file selected
- After upload:
  - Source appears in "Uploaded Files" list
  - Model's word count updates in UI
  - "Analyze Style" button enabled when word count >= minimum
- Uploaded files list shows:
  - File name
  - Word count per file
  - Upload date

**Database Operations:**
- Files saved to: /server/data/human-model-sources/{id}-{filename}
- human_model_sources table records:
  - id (UUID), human_model_id (FK), file_name
  - file_path (disk location), file_type, word_count
  - uploaded_at timestamp
- human_models table updated:
  - total_word_count incremented by uploaded file's word count
  - updated_at timestamp refreshed
