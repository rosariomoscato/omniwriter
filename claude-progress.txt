#### Feature #107: Password reset flow ✅
**Status:** PASSING

**Implementation Summary:**

Implemented complete password reset flow with forgot and reset functionality.

**Backend Changes (server/src/routes/auth.ts):**
- Updated `POST /api/auth/forgot-password` endpoint:
  - Generates unique reset token using UUID
  - Stores token in `password_reset_tokens` table
  - Token expires in 1 hour
  - Logs reset URL to console (development mode)
  - Always returns success to prevent email enumeration

- Added `POST /api/auth/reset-password` endpoint:
  - Validates reset token (must exist, unused, not expired)
  - Validates new password (8+ chars, uppercase, lowercase, number)
  - Hashes new password with bcrypt
  - Marks token as used
  - Invalidates all existing user sessions (forces re-login)
  - Returns success message

**Database Changes (server/src/db/database.ts):**
- Added `password_reset_tokens` table:
  - id (PRIMARY KEY)
  - user_id (FK to users)
  - token (unique reset token)
  - expires_at (token expiration timestamp)
  - used (boolean flag, default 0)
  - created_at (timestamp)
- Added indexes for user_id and token columns

**Frontend Components Created:**

1. **ForgotPasswordPage (client/src/pages/ForgotPasswordPage.tsx):**
   - Email input form
   - Success state with instructions
   - Development mode notice (check terminal for link)
   - Link back to login

2. **ResetPasswordPage (client/src/pages/ResetPasswordPage.tsx):**
   - Token validation from URL query params
   - New password input with show/hide toggle
   - Confirm password input
   - Real-time password strength indicators:
     - 8+ characters
     - 1 uppercase letter
     - 1 lowercase letter
     - 1 number
   - Password matching validation
   - Success state with auto-redirect to login

**App Routes (client/src/App.tsx):**
- Added `/forgot-password` route → ForgotPasswordPage
- Added `/reset-password` route → ResetPasswordPage
- Public routes (no authentication required)

**Verification Steps:**
1. User clicks "Password dimenticata?" link on login page
2. User enters email and submits
3. Backend generates reset token and stores in DB
4. Reset link logged to console (development mode)
5. User visits reset link with token
6. User enters new password (with validation)
7. Backend validates token and updates password
8. All user sessions invalidated
9. User redirected to login with new password
