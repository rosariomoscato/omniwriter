### Feature #405: Enforcement limite spazio su upload

**Objective:** Impedire il caricamento di file quando l'utente ha raggiunto o supererebbe il limite di 100 MB. Restituire un errore chiaro che spiega quanto spazio è disponibile.

**Implementation:**

#### Modified Files:

1. **server/src/routes/sources.ts** (3 upload routes updated)
   - Added imports: `getUserStorageInfo`, `hasStorageQuota`
   - Updated POST /sources/upload (standalone upload)
   - Updated POST /projects/:id/sources/upload (project upload)
   - Updated POST /sagas/:id/sources/upload (saga upload)
   - All routes now check `hasStorageQuota()` before accepting files
   - Returns 413 status with `STORAGE_QUOTA_EXCEEDED` error code
   - Error response includes:
     - `used`: current bytes used
     - `limit`: storage limit bytes
     - `available`: remaining bytes
     - `required`: file size bytes
     - `usedMB`, `limitMB`, `availableMB`, `fileMB` (MB values for display)
   - Cleans up uploaded file if quota exceeded

2. **server/src/routes/human-models.ts** (upload + delete updated)
   - Added imports: `getUserStorageInfo`, `hasStorageQuota`, `increaseUserStorage`, `decreaseUserStorage`
   - Updated POST /api/human-models/:id/upload
   - Added storage quota check before upload
   - Added `increaseUserStorage()` call after successful upload
   - Updated DELETE /api/human-models/:id/sources/:sourceId
   - Added `decreaseUserStorage()` call when file is deleted
   - Tracks file size from filesystem for accurate storage updates

**Error Message Format:**
```
HTTP 413 Payload Too Large
{
  "message": "Spazio insufficiente. Hai usato 95 MB su 100 MB (5 MB disponibili). Il file richiede 10 MB.",
  "code": "STORAGE_QUOTA_EXCEEDED",
  "used": 99614720,
  "limit": 104857600,
  "available": 5242880,
  "required": 10485760,
  "usedMB": 95,
  "limitMB": 100,
  "availableMB": 5,
  "fileMB": 10
}
```
