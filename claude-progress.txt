### Feature #445: Fix dotenv config order for Google OAuth
**Status:** COMPLETED and PASSING

**Root Cause:**
- In `server/src/index.ts`, `dotenv.config()` was called AFTER the import statements
- When `authRouter` was imported, it immediately configured the Google OAuth Strategy
- At that point, `process.env.GOOGLE_CLIENT_ID` and `process.env.GOOGLE_CLIENT_SECRET` were undefined
- This caused Google OAuth to fail in production with "invalid client" errors

**Fix Applied:**
- Changed `import dotenv from 'dotenv'` + `dotenv.config({ path: envPath })` to `import 'dotenv/config'` as the FIRST import
- This ensures environment variables are loaded before any module that uses them
- Removed the redundant `dotenv.config()` call that was after all imports
- Added explanatory comment about the importance of import order

**Verification:**
- TypeScript compilation succeeds without errors
- Server starts successfully with the new import order
- Health check returns healthy status
- In compiled dist/index.js: `require("dotenv/config")` is at line 41, `require("./routes/auth")` is at line 49
- Git Commit: d2b062a

### Feature #438: Correggere risposta API marketplace admin quando vuoto
**Status:** COMPLETED and PASSING

**Root Cause:**
- Express route ordering bug: `GET /api/marketplace/:id` was defined before `GET /api/marketplace/admin`
- Express matched 'admin' as the `:id` parameter, querying DB for `id='admin'`, finding nothing, returning 404 "Marketplace item not found"
- Frontend showed misleading error "Impossibile caricare le opere" when marketplace was simply empty

**Fix Applied:**
- Moved `GET /admin` route definition BEFORE `GET /:id` in server/src/routes/marketplace.ts
- Added comment explaining the route ordering requirement
- Removed duplicate old route definition that was after /:id

**Verification:**
- Without auth: returns 401 "Authentication required" (not 404)
- With admin auth + items: returns `{ items: [...], pagination: {...} }` with 200
- With admin auth + no matching items: returns `{ items: [], pagination: { page: 1, limit: 20, total: 0, totalPages: 0 } }` with 200
- Frontend admin marketplace page loads correctly, shows stats and table
- Zero console errors
- No mock data patterns found
