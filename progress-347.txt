### Feature #347: Fix: Word count e data ultima modifica non visibili nelle card

**Objective:** Il conteggio parole e la data dell'ultima modifica non appaiono nelle card dei progetti nella dashboard, anche quando i toggle sono attivi nel popup "Personalizza".

**Root Causes:**
1. Il campo `word_count` della tabella `projects` viene aggiornato SOLO durante la generazione automatica dei capitoli
2. Se i capitoli vengono modificati manualmente nell'editor, il conteggio totale del progetto non viene ricalcolato
3. La condizione `project.word_count > 0` non è mai vera perché il valore nel database è 0

**Solution:**
1. Modificare la query SQL in `GET /api/projects` per calcolare il word_count totale dalla somma dei word_count dei capitoli
2. Aggiornare il `updated_at` del progetto quando un capitolo viene salvato

**Implementation:**

#### 1. Backend - Calculate word_count from chapters (server/src/routes/projects.ts)

Modified the projects query to include a LEFT JOIN with chapters and calculate the sum:
```sql
SELECT
  p.*,
  GROUP_CONCAT(pt.tag_name, ',') as tags,
  COALESCE(SUM(c.word_count), 0) as calculated_word_count
FROM (${query}) as p
LEFT JOIN project_tags pt ON p.id = pt.project_id
LEFT JOIN chapters c ON p.id = c.project_id
GROUP BY p.id
ORDER BY p.updated_at DESC
```

Updated the mapping to use calculated_word_count:
```typescript
const projectsWithTags = (projects as any[]).map(project => ({
  ...project,
  tags: project.tags ? project.tags.split(',').filter((t: string) => t) : [],
  word_count: project.calculated_word_count || 0
}));
```

#### 2. Backend - Update project's updated_at when chapter is saved (server/src/routes/chapters.ts)

Added project update after chapter save:
```typescript
// Feature #347: Update project's updated_at when a chapter is saved
db.prepare(`
  UPDATE projects SET updated_at = datetime('now') WHERE id = ?
`).run(existingChapter.project_id);
```

Also updated the SELECT query to include `project_id` for the existing chapter.
